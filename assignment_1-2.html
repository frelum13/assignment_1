Assignment 1: Basic Concepts of Object-Oriented Programming | 15 Points

Introduction

The overall goal of Assignment 1 is to practice the basics of object-oriented programming.
In this assignment, you will create a system for exploring datasets. The system is able to load and store a dataset, and it includes functionality for exploring the dataset, such as plotting columns and calculating simple metrics.

Datasets

In this assignment, we will work with real-life data. You can choose between the two datasets described in this section. You can also choose to work with both datasets - in this case, you will receive bonus points (see Section “Bonus Points” below).
Hint: If you design your system well, working with both datasets will be very little extra work compared to working with only one.

Heart Disease Dataset

This dataset contains data on heart disease diagnosis. It contains 15 14 columns, including one column (the column target) that indicates the presence (or absence) of heart disease in the patient. Missing values in this dataset are specified with ? in the data files.

This dataset was created by the following institutions and principal investigators:

  1. Hungarian Institute of Cardiology. Budapest: Andras Janosi, M.D.
  2. University Hospital, Zurich, Switzerland: William Steinbrunn, M.D.
  3. University Hospital, Basel, Switzerland: Matthias Pfisterer, M.D.
  4. V.A. Medical Center, Long Beach and Cleveland Clinic Foundation: Robert Detrano, M.D., Ph.D.

For more information on this dataset, see https://archive.ics.uci.edu/ml/datasets/heart+disease.

Parkinson Speech Dataset

This dataset was created by Sakar et al. [1] and contains data from speech recordings of subjects with Parkinson’s disease as well as healthy subjects. It contains 31 columns, including one column (the column class) that indicates the presence (or absence) of Parkinson’s disease and columns containing time-frequency-based features extracted from voice samples. Missing values in this dataset are specified with n/a in the data files.

For more information on this dataset, see Sakar et al. [1] and
https://archive.ics.uci.edu/ml/datasets/Parkinson+Speech+Dataset+with++Multiple+Types+of+Sound+Recordings.

Dataset Sources

Both datasets have been made available in the UCI Machine Learning Repository [2].

[1] Sakar, B. E., et al. (2013). Collection and analysis of a Parkinson speech dataset with multiple types of sound recordings. IEEE Journal of Biomedical and Health Informatics, 17(4), 828-834.
[2] Dua, D. and Graff, C. (2019). UCI Machine Learning Repository [http://archive.ics.uci.edu/ml]. Irvine, CA: University of California, School of Information and Computer Science.

General Functionality of the System

Input Files

All input files that you need for this assignment are provided in the “Assignment 1” Section in the TeachCenter.

Data Files

The data for a dataset is contained in several .csv files. These .csv files have no column headers; the column names can be found in an extra .json file (in the same directory as the .csv files), along with a description of each column.

Config File

For each dataset, we have provided a .json template for a config file. The config file contains information about how the different data columns should be plotted by the system (for more information, see Section “Plotter Classes”. Furthermore, the config file contains information on the dataset that will be used by the class Dataset (for more information, see Section “Class Dataset”).
For example, the provided template for the Heart Disease Dataset looks like this:

{
    "plot_types": {
        "age": "",
        "sex": "",
        "cp": "",
        "trestbps": "",
        "chol": "",
        "fbs": "",
        "restecg": "",
        "thalach": "",
        "exang": "",
        "oldpeak": "",
        "slope": "",
        "ca": "",
        "thal": "",
        "target": ""
    },
    "target_feature": "target",
    "column_info_file": "column_info_heart_disease.json",
    "data_folder": "heart_disease_data",
    "dataset_name": "Heart Disease Dataset",
    "na_characters": ["?"],
    "binarize_threshold": 1
}

It is up to you to decide which data column should be plotted with which plot type. In your system, the following plot types should be available: box, bar, and histogram. When you have decided which plot type to use for each data column, fill in the empty strings in the config file (e.g., "age": "box").

Loading Data

In the data folder specified in the configuration file, there are several .csv files that contain the data for the respective dataset. Suggestion for second sentence: The Dataset class (see Section “Class Dataset”) will be responsible for loading, combining, and storing the data contained in these files.

Plotting and Calculating Metrics

After a suitable data structure has been found, the next step is to take a closer look at the data. Arbitrary features will be plotted and simple metrics will be calculated to gain further insight into the data.
Note that in this assignment, the word “feature” simply means a column in the dataset.

Class Dataset (5P)

The class Dataset represents a dataset. It holds the data along with a description of the data. It provides functionality for loading data files, writing data to a file, and other dataset-related functionality.

Required Properties

Name Type Getter Setter Description
name str yes yes The name of the dataset
column_info dict[str, str] yes no Dictionary with the column names as keys and the column descriptions as values (this information is available in the .json-file specified under "column_info_file" in the configuration dictionary)
feature_names list[str] yes no List of feature names
target_feature_name str yes no The name of the target feature

Required Method: __init__

  • Functionality:
    __init__ receives a dictionary containing information about a dataset (obtained from a .json config file, see Section “Config File”). The requested dataset is read (and stored) during object initialization. The target feature is binarized, if "binarize_threshold" is set in the configuration. The configuration dictionary contains the following entries:

    Name Type Description
    "dataset_name" str The name of the dataset
    "data_folder" str The path to the folder containing .csv-files to read
    "column_info_file" str The path (relative to the "data_folder") to the .json-file containing information on the different columns in the data files
    "target_feature" str The name of the target feature
    "na_characters" list[str] Which characters in data files should be treated as NaN
    "binarize_threshold" int [Optional] If specified, all target labels less than "binarize threshold" are set to 0, all labels equal or greater are set to 1

  • Parameters:

    Name Type Description
    config_info_dict dict A dictionary containing all relevant infos to instantiate a Dataset object

Required Method: __str__

  • Functionality:
    This method returns a string in the format

    "Dataset Info:\n\tDataset name: <name of the dataset>\n\tNumber of features: <number of columns>, number of rows: <number of rows>"

    where <name of the dataset> is the name of the dataset that was defined in the configuration, <number of columns> is the number of columns in the data, and <number of rows> is the number of rows in the data.

    So, printing a Dataset object could show:

    Dataset Info:
    Dataset name: Test Dataset
    Number of features: 4, number of rows: 10

  • Return:

    Type Description
    str See description above

Required Method: write_data_to_file

  • Functionality:
    This method writes the data to a .csv file (only the data, not the column headers/names). The column order is equal to the column order of the original files.

  • Parameters:

    Name Type Description
    output_file str The name of the output file

Required Method: get_feature_values

  • Functionality:
    This method returns the values from the requested feature(s).
    If a single feature name was passed, a single list - containing the values from that feature - is returned. If multiple feature names are passed, multiple lists are returned as a list of lists. Depending on the parameter remove_missing, rows with missing values are included or omitted (for details, see parameter description below).

  • Parameters:

    Name Type Default Value Description
    *feature_names str Arbitrary many feature names as strings (see also here)
    remove_missing bool False If True, remove values for each row where at least one of the features in feature_names has a missing value. If False, return all values, including missing values.

  • Return:

    Type Description
    list / list[list] A list of feature values if only one feature was specified in feature_names. A list of lists (each list contains the values for one feature) if more than one feature was specified in feature_names. The order of the lists is the same as in feature_names.

Required Method: get_target_values

  • Functionality:
    This method returns the values from all rows of the target column as a list.
  • Return:
    Type Description
    list A list containing the target values.

Abstract Class Plotter (1P)

The class Plotter is an abstract class that defines the interface for any concrete plotters.

Required Class Method: get_plotter_classes

  • Functionality:
    This class method returns a dictionary that contains information about all subclasses of Plotter.

  • Return:

    Type Description
    dict[str, type] A dictionary where each key is a plotter name and its value is the type of the corresponding plotter class. E.g., one dictionary entry will be "bar": <class 'plotter.BarPlotter'>

Required Static Abstract Method: plot_feature

  • Parameters:
    Name Type Description
    feature_name str The name of the feature to plot
    feature_values list The values of the feature specified by feature_name

Hint: A static abstract method is a method of a class that is both a staticmethod and an abstractmethod. You will have to use both decorators in the right order.

Required Static Abstract Method: plot_feature_vs_target

  • Parameters:
    Name Type Description
    feature_name str The name of the feature to plot
    feature_values list The values of the feature specified by feature_name
    target_name str The name of the target variable
    target_values list The values of the target variable

Plotter Classes (4.5P)

The plotter classes are subclasses of the class Plotter. Write three plotter classes:

  1. BarPlotter
  2. HistogramPlotter
  3. BoxPlotter

The BarPlotter can plot bar charts (where every bar represents how often a certain value of a feature occurs), the HistogramPlotter can plot histograms, and the BoxPlotter can plot box plots.

These classes implement the methods plot_feature and plot_feature_vs_target.

Required Static Method: plot_feature

  • Functionality:
    This method visualizes a single feature. It creates a plot of the feature values (depending on which plotter class the method is in, it will be either a bar chart, a histogram, or a box plot). Design the layout of the plots yourself.
    The parameters of this method are defined in the abstract Plotter class. The return value is a figure-object containing the plot (use your preferred plotting library).

  • Return:

    Type Description
    matplotlib.figure.Figure or
    plotly.graph_objects.Figure or
    holoviews.Layout or
    holoviews.Overlay    		 A figure-object containing the plot

Required Static Method: plot_feature_vs_target

  • Functionality:
    This method visualizes a feature in combination with the target variable. It creates a plot that shows the feature values separated according to the corresponding values of the target variable. Design the plots yourself: For example, the BarPlotter could create a grouped bar chart (grouped by the value of the target variable), or it could create several bar charts next to each other (one for each target value, i.e., two in the case of a binary target variable).
    The parameters of this method are defined in the abstract Plotter class. The return value is a figure-object containing the plot (use your preferred plotting library).

  • Return:

    Type Description
    matplotlib.figure.Figure or
    plotly.graph_objects.Figure or
    holoviews.Layout or
    holoviews.Overlay    		 A figure-object containing the plot

Class DatasetExplorer (4P)

The class DatasetExplorer contains methods for exploring the dataset, such as plotting features and calculating simple metrics.

Required Method: __init__

  • Functionality:
    The __init__ initializes a DatasetExplorer object.

  • Parameters:

    Name Type Description
    dataset Dataset A dataset object created with the Dataset class
    plot_types dict[str, str] A dictionary taken from the configuration file of the respective dataset mapping each feature to the name of a Plotter type. It could look like this:
    {"age": "box", "sex": "bar"}

Required Method: __str__

  • Functionality:
    This method returns a string in the format

    "DatasetExplorer Info:\n\tDataset name: <name of the dataset>"

    So, printing a DatasetExplorer object could show:

    DatasetExplorer Info:
    Dataset name: Test Dataset
  • Return:
    Type Description
    str See description above

Required Method: print_feature_names

  • Functionality:
    Prints the names of the features contained in the dataset seperated by a comma that is followed by a space (i.e. ", "):
    Feature names:
<name of feature 1>, <name of feature 2, ... , <name of feature N>
    For example, the output for a dataset containing two features, age and sex, looks like this:
    Feature names:
age, sex

Required Method: print_feature_description

  • Functionality:
    This method either prints the description of one requested feature or the description of all features. The features’ descriptions are found in the column_info.json file of the respective dataset.

    In the case where the description of only one feature is requested, print:
    <feature name>: <feature description>

    Otherwise, print:

    Features and descriptions:
    <feature 1 name>: <feature 1 description>
    <feature 2 name>: <feature 2 description>
    ...
    <feature N name>: <feature N description>

    Again, with a dataset comprising two features, age and sex, the output of this method (called without feature_name) is as follows:

    Features and descriptions:
    age: age in years
    sex: sex (1 = male; 0 = female)

  • Parameters:

    Name Type Default Value Description
    feature_name NoneType/str None If feature_name is None, print the description for all the features contained in the dataset. If feature_name is set to a specific feature name, print the description for that feature only.

Required Method: plot_feature

  • Functionality:
    plot_feature delegates plotting to the appropriate Plotter's plot_feature static method, creating appropriate arguments via calls to methods of the Dataset to explore. It returns the figure created by the Plotter.

  • Parameters:

    Name Type Description
    feature_name str The name of the feature to plot

  • Return:

    Type Description
    matplotlib.figure.Figure or
    plotly.graph_objects.Figure or
    holoviews.Layout or
    holoviews.Overlay    		 A figure-object containing the plot

Required Method: plot_feature_vs_target

  • Functionality:
    plot_feature_vs_target delegates plotting to the appropriate Plotter's plot_feature_vs_target static method, creating appropriate arguments via calls to methods of the Dataset to explore. It returns the figure created by the Plotter.

  • Parameters

    Name Type Description
    feature_name str The name of the feature to plot

  • Return:

    Type Description
    matplotlib.figure.Figure or
    plotly.graph_objects.Figure or
    holoviews.Layout or
    holoviews.Overlay    		 A figure-object containing the plot

Required Method: calculate_correlation_between_features

  • Functionality:
    Calculate and return the correlation coefficient and p-value using one of the following methods (depending on the parameter method):

    • pearson
    • spearman
    • kendalltau

  • Parameters:

    Name Type Default Value Description
    feature_name_1 str The name of the first feature
    feature_name_2 str The name of the second feature
    method str "pearson" The method used to calculate the correlation between the two features (can be either "pearson", "spearman" or "kendalltau")

  • Return:

    Type Description
    tuple The correlation coefficient r and the p-value as a tuple in the form of (r, p)

Note that you are allowed to use scipy. Make sure that you properly handle the case in which one or more values of a feature are missing.

Required Method: print_quartiles_of_feature

  • Functionality:
    This method prints the quartiles of the specified feature in the following format:

    Quartiles:
    Q1: <first quartile>
    Q2: <second quartile>
    Q3: <third quartile>

    For example:

    Quartiles:
    Q1: 8.0
    Q2: 15.0
    Q3: 18.5

  • Parameters:

    Name Type Description
    feature_name str The name of the feature for which the quartiles should be calculated and printed
